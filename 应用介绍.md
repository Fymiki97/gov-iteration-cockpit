# 政务产研迭代进度看板 — 应用介绍

## 一、应用定位

这是一款面向**政务产品研发团队**的 **B 端迭代管理看板**，部署在 **WPS 365 智能应用平台**上。

核心定位是：把分散在 WPS 多维表格里的需求、里程碑、风险数据，汇总成**一张可实时刷新的管理驾驶舱**，让产研负责人、项目经理和干系人不用翻表，就能掌握 2026 年政务产品研发迭代的整体进度与风险态势。

风格上按政务系统习惯设计：简洁、专业、规整，适合 PC 端日常查看和汇报。

---

## 二、建设背景

| 维度 | 说明 |
|------|------|
| **业务背景** | 2026 年政务产品研发迭代规划，需求按月度排期（2&3 月～12 月），需跨月追踪进度、里程碑和风险 |
| **数据现状** | 原始数据维护在 WPS 多维表格（dbsheet）中，分三张表：需求、里程碑、风险 |
| **痛点** | 表格适合录入，但不适合「一眼看全局」；月度进度、风险状态、里程碑节点分散，汇报和决策成本高 |
| **解决思路** | 基于 WPS365 Open SDK 读取多维表格，在前端做聚合、加权计算和可视化展示，数据仍由表格维护，看板只读展示 |

数据来源绑定同一份多维表格（`FILE_ID: Dm5Wx1ph11MNih2SbwZurxjFLUZTboQEF`）：

- **Sheet 21**：需求清单（最多 2000 条）
- **Sheet 23**：里程碑（最多 200 条）
- **Sheet 24**：风险项（最多 50 条）

## 三、核心功能

应用首页即看板，包含 **4 个 Tab**：

### 1. 迭代概览

全局驾驶舱，一屏掌握整体情况：

- **需求总数 / 已完成 / 剩余风险 / 完成进度** 四张核心指标卡
- **完成进度**采用状态加权算法（非简单计数），公式：`Σ(各需求状态进度) ÷ 需求总数`，权重覆盖从「未开始」到「已发布」全阶段
- **状态分布柱状图**，支持 hover 查看各状态数量
- **风险跟踪**：按应对策略分组，展示未关闭/未规避的活跃风险，并关联 ONES 需求
- 指标卡可点击跳转需求列表，带筛选条件

### 2. 月度迭代情况

按迭代月份（2&3 月～12 月）查看：

- **「全部」视图**：各月卡片概览，含进度、状态分布标签、里程碑数、风险数
- **单月详情视图**：
  - 进度（含权重说明 Popover）
  - 状态分布**饼图 + 数值表**（悬浮可看具体数量）
  - 里程碑进展（已完成/延期完成自动折叠）
  - 风险跟踪、需求列表
  - 里程碑完成数、未关闭风险数等细分指标

### 3. 需求列表

全量需求检索与筛选：

- 多维度筛选：迭代月份、状态、负责人
- 全文搜索：标题、ONES ID、状态、项目、负责人等
- 分页展示（每页 20 条）
- 支持从概览页带条件跳转（已完成、风险关联等）
- 每条需求可跳转 ONES 链接

### 4. 里程碑

两种视图模式：

- **选具体月份**：时间轴样式，展示该月里程碑节点、计划/实际时间、延期情况
- **选「全部」**：**自然时间视图**（近期优先）
  - 按「已逾期 → 今日 → 明日 → 本周 → 本月 → 下月 → 已完成」分组
  - 顶部高亮**下一个待处理里程碑**
  - 快捷筛选（全部 / 近 7 天 / 待完成）+ 名称搜索
  - 列表标注所属迭代月份
  - 仅有计划时间的条目才算里程碑

## 五、数据与价值

### 管理的数据维度

| 数据类型 | 主要字段 | 统计能力 |
|---------|---------|---------|
| **需求** | 标题、状态、优先级、项目、迭代月份、开发/测试负责人、ONES ID、排期 | 按状态加权进度、按月聚合、完成率 |
| **里程碑** | 名称、计划时间、实际时间、状态、所属迭代、备注 | 逾期判断、时间分组、完成/待办统计 |
| **风险** | 事项、状态、提报日期、归属产品、迭代、应对策略、关联 ONES | 活跃风险计数、策略分组、需求关联 |

### 核心计算逻辑

- **进度权重**：按需求所处阶段映射累计进度（如「开发中」40%、「测试中」77%、「已发布」100%）
- **风险判定**：排除「已解除 / 已关闭 / 已完成」后，其余（含「未规避」「未关闭」）计为活跃风险
- **里程碑有效性**：必须有计划时间才算里程碑
- **工作日计算**：里程碑延期天数考虑 2026 年法定节假日与调休

### 业务价值（预期成效）

- **汇报效率**：从翻表汇总 → 打开看板即得全局数据
- **风险可见**：未关闭风险集中展示，避免遗漏
- **里程碑聚焦**：「全部」视图按自然时间排序，近期节点置顶
- **数据一致性**：看板只读表格，维护仍在原表，无需双写

> 说明：代码库中没有接入访问量、用户活跃度等运营埋点，上述为产品设计目标。若需对外汇报「上线后使用情况」，需在 WPS 平台侧补充访问统计或自行埋点。

## 六、使用与获取方式

### 线上使用（已部署）

访问 WPS 365 智能应用地址：

**https://o.wpsgo.com/app/41000207/9w1qvC8NVtKs/**

首次使用会触发 WPS OAuth 授权（需 `kso.dbsheet.readwrite` 权限），授权后即可读取多维表格数据。页面右上角「刷新数据」可手动重新拉取。

### 本地开发与预览

**环境要求**：Node.js ≥ 20、pnpm

```bash
# 1. 安装依赖
pnpm install

# 2. 启动开发环境（前端 5917 + 后端 4917）
pnpm run dev

# 3. 浏览器访问
http://localhost:5917
```

本地开发时 `/api/*` 自动代理到后端，OAuth 回调地址为 `http://localhost:5917/api/oauth/callback`。

### 打包部署

```bash
# 类型检查 + 代码检查
pnpm run check:types
pnpm run lint

# 打包（生成到 .publish/ 目录）
pnpm run pack
# → .publish/dist-frontend.zip  （前端静态资源）
# → .publish/dist-server.zip    （后端 Node 服务）
```

将两个 zip 上传到 **WPS 365 智能应用平台**对应的前端/后端运行时，即可更新线上版本。`pnpm run pack` 不会自动发布，需手动上传。

### 获取源码

- **项目 ID**：`760386581358207`（WPS Comate 工作区）
- **代码仓库**：当前 Git 工作区，主入口为 `client/src/pages/DashboardPage.tsx`
- **权限范围**：应用申请 `kso.dbsheet.readwrite`，用于读取绑定的多维表格
